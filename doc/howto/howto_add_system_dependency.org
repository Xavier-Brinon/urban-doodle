#+title: How to Add a System Dependency to a Railpack Build
#+author: Xavier Brinon
#+date: [2026-03-26 Thu]
#+startup: indent
#+options: toc:t num:t ^:{}

* Goal

Add a system-level package (e.g. Pandoc, FFmpeg, ImageMagick) to a
Railpack build so it is available during the build phase.

* Prerequisites

- A project with a =railpack.json= file (or willingness to create one)

* Steps

** Find the apt package name

Search [[https://packages.debian.org][Debian packages]] online. Common
packages:

| Tool        | Apt package name |
|-------------+------------------|
| Pandoc      | =pandoc=         |
| FFmpeg      | =ffmpeg=         |
| ImageMagick | =imagemagick=    |
| Git         | =git=            |
| curl        | =curl=           |

** Add the package to =railpack.json=

#+begin_src json
{
  "$schema": "https://schema.railpack.com",
  "buildAptPackages": ["pandoc"]
}
#+end_src

For multiple packages:

#+begin_src json
{
  "$schema": "https://schema.railpack.com",
  "buildAptPackages": ["pandoc", "ffmpeg"]
}
#+end_src

** If the package is needed at runtime too

=buildAptPackages= only installs during build. For the final image:

#+begin_src json
{
  "buildAptPackages": ["pandoc"],
  "deploy": {
    "aptPackages": ["pandoc"]
  }
}
#+end_src

** Rebuild

#+begin_src bash
railpack build .
#+end_src

* Verification

Check the build logs for the =install apt packages= step:

#+begin_example
#9 install apt packages: pandoc
...
#9 Setting up pandoc (2.17.1.1) ...
#+end_example

For runtime packages, verify inside the image:

#+begin_src bash
podman run --rm --entrypoint sh my-app -c "pandoc --version"
#+end_src

* Troubleshooting

| Problem                     | Solution                                             |
|-----------------------------+------------------------------------------------------|
| Package not found           | Check exact name at [[https://packages.debian.org][packages.debian.org]] |
| Wrong version               | Railpack uses Debian Bookworm; version is fixed      |
| In build but not runtime    | Add to =deploy.aptPackages= as well                  |
